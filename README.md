# rts-doc-search — поиск документов по адресу

Программа с окном (GUI) для поиска документов, относящихся к одному адресу,
сразу в нескольких папках-библиотеках адресов с **разной структурой**.

## Какие структуры папок понимает

**Структура A — с папками-буквами:**

```
Корневая папка/
  А/  Б/  В/ ...           ← папки-буквы алфавита
    <улица>/               ← улицы на эту букву
      <дом>/               ← номера домов
        <квартира>/        ← номера квартир
          документы...     ← файлы по этому адресу
```

**Структура B — улицы идут сразу:**

```
Корневая папка/
  <улица>/
    <дом>/
      <квартира>/
        документы...
```

Программа **сама определяет** структуру каждой папки — указывать ничего не нужно.
Можно подключить любое количество папок (в задаче их 3), в том числе сетевые
(пути вида `\\сервер\папка` или подключённые сетевые диски).

## Как пользоваться

1. Запустить программу (см. ниже).
2. В блоке **«Папки с адресами»** нажать **«Добавить…»** и указать каждую из
   корневых папок. Список запоминается между запусками.
3. Ввести **Улицу** (можно часть названия), при желании уточнить **Дом** и
   **Квартиру**.
4. Нажать **«Искать»**.
5. В списке появятся все найденные документы. Двойной клик открывает файл,
   кнопка **«Открыть папку с файлом»** — папку, где он лежит.

Поиск устойчив к мелочам: регистр не важен, `ё` = `е`, понимает приставки
«ул.», «д.», «кв.». По улице ищет по части названия, по дому и квартире —
точное совпадение номера.

## Запуск

Нужен установленный **Python 3** (на Windows при установке отметьте галочку
«tcl/tk and IDLE» — модуль Tkinter входит в стандартную поставку).

```
python address_search.py
```

На Windows можно просто дважды кликнуть по файлу `address_search.py`.

Дополнительные библиотеки **не требуются** — используется только стандартная
библиотека Python.

## Где хранятся настройки

Список папок сохраняется в файле `.address_search_config.json` в домашней
папке пользователя.
